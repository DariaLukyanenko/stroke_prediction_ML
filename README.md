# Предсказание инсульта по табличным медицинским данным

## Описание
Блокнот посвящён задаче бинарной классификации риска инсульта по клиническим признакам пациентов.  
В работе сочетаются разведочный анализ данных, feature engineering, борьба с дисбалансом классов, контроль утечки данных и сравнение нескольких моделей машинного обучения и нейросетей.

## Данные
Используется табличный медицинский датасет пациентов с целевой переменной `stroke`.

Основные признаки:
- пол;
- возраст;
- гипертензия;
- сердечно-сосудистые заболевания;
- семейное положение;
- тип занятости;
- тип проживания;
- средний уровень глюкозы;
- BMI;
- статус курения.

Особенности данных:
- присутствуют пропуски в `bmi`;
- классы сильно несбалансированы;
- соотношение классов примерно **19:1** в пользу отсутствия инсульта.

## Что сделано в блокноте
- загрузка и первичный анализ датасета;
- заполнение пропусков в `bmi` средним значением;
- анализ числовых и категориальных признаков;
- кодирование категориальных признаков;
- создание дополнительных групповых признаков;
- балансировка выборки через **SMOTE**;
- разбор проблемы **data leakage**;
- отбор признаков с помощью:
  - `Mutual Information`;
  - `Chi-Squared`;
  - `ANOVA`;
- масштабирование признаков;
- обучение и сравнение нескольких моделей;
- проверка качества на данных с утечкой и без неё;
- интерпретация признаков через AutoML и SHAP;
- дополнительные визуализации разделяющих поверхностей для разных алгоритмов классификации.

## Модели
В блокноте рассматриваются:
- **XGBoostClassifier**;
- **LightGBM**;
- **Logistic Regression**;
- логистическая регрессия на **PyTorch**;
- двухслойная полносвязная сеть на **PyTorch**;
- прототип **TabFPN / TabPFN**;
- **LightAutoML** для табличной классификации.

## Подготовка данных
Ключевые шаги подготовки:
- удаление `id`;
- заполнение пропусков;
- label encoding и one-hot encoding;
- балансировка классов;
- разделение данных на train / test / validation;
- отдельный анализ сценариев:
  - **без утечки данных**;
  - **с утечкой данных**.

## Метрики
Для оценки моделей используются:
- accuracy;
- ROC-AUC;
- precision;
- recall;
- F1-score;
- cross-validation score.

## Основные результаты

### Классические модели без утечки данных
- **XGBoost**

- **Logistic Regression**

  - recall по классу инсульта: **0.65**

## Вывод
Блокнот показывает полный пайплайн решения задачи классификации на табличных медицинских данных: от EDA и feature engineering до сравнения бустингов, линейных моделей и нейросетей.  
Ключевой акцент сделан на двух вещах:
- сильный дисбаланс классов;
- влияние утечки данных на качество модели.

Лучшие результаты на подготовленных данных показывают табличные ансамблевые и foundation-подходы, но на реальной несбалансированной валидации качество по положительному классу остаётся ограниченным, что важно учитывать в медицинских задачах.
